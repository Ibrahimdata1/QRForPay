# คู่มือใช้งาน QRForPay POS

## เข้าสู่ระบบ
1. เปิดแอพ → กรอกอีเมลและรหัสผ่าน → กด **"เข้าสู่ระบบ"**
2. แอพพาเข้าหน้าหลักทันที

แถบด้านล่างมี 5 แท็บ: **แดชบอร์ด / ออเดอร์ / สินค้า / โต๊ะ / ตั้งค่า**

---

## จัดการโต๊ะ (แท็บ "โต๊ะ")

### สีสถานะโต๊ะ

| สี | สถานะ | ความหมาย |
|----|-------|---------|
| 🟢 เขียว | ว่าง | ไม่มีออเดอร์ |
| 🟠 ส้ม | รอทำ | มีออเดอร์รออยู่ ยังไม่ได้ทำ |
| 🟣 ม่วง | กำลังทำ | ครัวกำลังปรุงอาหาร |
| 🔵 น้ำเงิน | พร้อมเสิร์ฟ | อาหารเสร็จ รอเสิร์ฟ/ชำระ |

### ดูรายละเอียดโต๊ะ
- กดที่การ์ดโต๊ะ → เห็นรายการออเดอร์และยอดรวม
- กด **"สร้าง QR"** เพื่อแสดง QR code ให้ลูกค้าสแกนสั่งอาหารเอง

### ยกเลิกรายการอาหาร (Per-Item Cancel)
- กดโต๊ะที่ต้องการ → กด **"ดูออเดอร์"**
- กดปุ่มยกเลิก (X) ข้างรายการที่ต้องการยกเลิก → ยืนยัน
- ยอดรวมจะอัพเดตอัตโนมัติ หากยกเลิกทุกรายการ ออเดอร์จะถูกยกเลิกทั้งหมด

---

## ระบบลูกค้าสั่งเองผ่าน QR

ลูกค้าสแกน QR บนโต๊ะ → เปิดเว็บเมนูบนโทรศัพท์ → สั่งและชำระเงิน PromptPay ได้เอง

**Flow ฝั่งพนักงาน:**
1. ไปแท็บ **"โต๊ะ"** → กดโต๊ะ → กด **"สร้าง QR"**
2. ตั้งจอหรือพิมพ์ QR วางบนโต๊ะ
3. ลูกค้าสั่งเอง → ออเดอร์ขึ้นที่แท็บ **"ออเดอร์"** ทันที (realtime) + แบนเนอร์แจ้งเตือน
4. พนักงานดูรายการ → กด "รับออเดอร์" → ทำ → กด "พร้อมเสิร์ฟ"
5. ลูกค้าสั่งเพิ่ม → สร้างออเดอร์ใหม่ทุกรอบ → แจ้งเตือนทุกครั้ง
6. คิดเงิน → ไปมุมมอง **"บิลรวมโต๊ะ"** → ยอดรวมทุกออเดอร์ → กดชำระ

**ถ้าลูกค้าจ่ายเงินสดแทน QR:**
- เปิดมุมมอง **"บิลรวมโต๊ะ"** → กด **"รับเงินสด"** → ยืนยัน (ชำระทุกออเดอร์ของโต๊ะพร้อมกัน)

**ฝั่งลูกค้า (เว็บ Vercel):**
- ตะกร้าแสดงรายการแยกสถานะ: เสิร์ฟแล้ว / กำลังเตรียม / รอดำเนินการ
- ยอดรวมอัพเดตอัตโนมัติทุก 10 วินาที + realtime เมื่อร้านแก้ไข
- ปุ่ม **"ชำระเงิน"** แสดงยอดรวมทั้งโต๊ะ + ลิงก์ "สั่งเพิ่ม" ด้านล่าง

---

## ดูออเดอร์ (แท็บ "ออเดอร์")

แท็บออเดอร์มี 2 มุมมอง สลับได้ที่แถบด้านบน:

### มุมมอง "ครัว"
- แสดงออเดอร์แต่ละรายการ เรียงตามเวลา
- ออเดอร์ใหม่จากลูกค้ามีป้าย **"ใหม่!"** สีแดง + สั่นเตือน (mobile)
- ค้นหาด้วยเลขออเดอร์ได้จากช่องค้นหา
- กรองสถานะ: ทั้งหมด / รอดำเนินการ / กำลังทำ / เสิร์ฟแล้ว
- กดการ์ดออเดอร์ → ดูรายละเอียด + ยกเลิกรายการ/ออเดอร์ได้

### มุมมอง "บิลรวมโต๊ะ"
- แสดงบิลรวมของแต่ละโต๊ะ (ออเดอร์ทุกรอบรวมกัน)
- เห็นทุกออเดอร์ของโต๊ะ + สถานะแต่ละออเดอร์
- **ยกเลิกรายเมนู**: กดปุ่ม X ข้างเมนูที่ต้องการยกเลิก (ถ้าอาหารทำแล้วจะมีคำเตือน)
- **ยกเลิกทั้งออเดอร์**: กดไอคอนถังขยะที่หัวออเดอร์
- รายการที่ยกเลิกแล้วแสดงขีดฆ่าสีจาง
- ยอดรวมคำนวณใหม่อัตโนมัติ + อัพเดตหน้าลูกค้าทันที
- กด **"ยืนยันรับโอน"** หรือ **"รับเงินสด"** เพื่อชำระบิลรวมทั้งโต๊ะ

### แจ้งเตือนออเดอร์ใหม่
- เมื่อลูกค้าสั่งอาหาร → แบนเนอร์สีแดงเด้งด้านบนสุดทุกหน้า + สั่น (mobile)
- แสดงเลขออเดอร์และหมายเลขโต๊ะ
- กดแบนเนอร์เพื่อปิด หรือรอ 6 วินาทีหายเอง

---

## รับเงิน QR PromptPay (ฝั่งพนักงาน)

1. เปิดออเดอร์ → กด **"ชำระเงิน"** → เลือก **QR PromptPay**
2. หน้าจอ QR เปิดขึ้น → ส่งมือถือให้ลูกค้าสแกน
3. ลูกค้าสแกนผ่านแอพธนาคาร → โอนเงิน
4. **กดปุ่มสีเขียว "ยืนยันรับเงินแล้ว"** เมื่อเห็นว่าลูกค้าจ่ายแล้ว

---

## รับเงินสด

1. เปิดออเดอร์ → กด **"รับเงิน"** → เลือก **เงินสด**
2. กรอกจำนวนเงินที่รับมา
3. แอพคำนวณเงินทอนให้อัตโนมัติ → กด **"ยืนยัน"**

---

## จัดการสินค้า (แท็บ "สินค้า" — เจ้าของร้านเท่านั้น)

- **เพิ่มสินค้า**: กดปุ่ม **+** มุมขวาล่าง → กรอกชื่อ ราคา สต๊อก หมวดหมู่ รูป → กด **"บันทึก"**
- **แก้ไข**: กดไอคอนดินสอ → แก้ → บันทึก
- **ลบ**: กดไอคอนถังขยะ → ยืนยัน

---

## ตั้งค่าร้าน (แท็บ "ตั้งค่า" — เจ้าของร้านเท่านั้น)

- **ชื่อร้าน / PromptPay ID**: แก้ไขและกด "บันทึก"
- **จำนวนโต๊ะ**: ตั้งจำนวนโต๊ะในร้าน → หน้า "โต๊ะ" จะแสดงตาม grid ที่ตั้งไว้
- **ธีม**: เลือก **สว่าง** หรือ **มืด** — ค่าจะถูกบันทึกข้ามการปิด-เปิดแอพ

---

## ปัญหาที่พบบ่อย

**QR สแกนแล้วแต่ระบบไม่ขึ้นสำเร็จ?**
→ กดปุ่มสีเขียว **"ยืนยันรับเงินแล้ว"** ได้เลย ไม่ต้องรอ

**ลูกค้าสแกน QR โต๊ะแต่เมนูไม่ขึ้น?**
→ ตรวจสอบสัญญาณอินเทอร์เน็ตของลูกค้า และตรวจว่า QR ถูก generate จากหน้า "โต๊ะ" ที่ถูกต้อง

**สถานะโต๊ะไม่อัพเดต?**
→ ลากหน้าจอลงจากด้านบน (pull to refresh) หรือออกแล้วกลับเข้าหน้าโต๊ะใหม่

**กดชำระเงินสดไม่ได้?**
→ ต้องกรอกจำนวนเงินที่รับมาก่อน และต้องไม่น้อยกว่ายอดที่ต้องจ่าย

**สินค้ากดไม่ได้?**
→ สต๊อกหมด ให้เจ้าของร้านอัปเดตสต๊อกในแท็บ "สินค้า"

---

## ขั้นตอนสรุป

```
ลูกค้าสั่งผ่าน QR:
สร้าง QR โต๊ะ → ลูกค้าสแกน → สั่งเอง → พนักงานดูที่ "ออเดอร์" → ทำ → เสิร์ฟ ✅

พนักงานขายเอง:
หน้าขาย → เพิ่มสินค้า → ตรวจตะกร้า → ชำระเงิน → ยืนยัน → เสร็จ ✅
```
