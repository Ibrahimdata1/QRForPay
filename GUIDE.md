# คู่มือ EasyShop POS — อ่านเข้าใจง่าย

> สำหรับเจ้าของร้าน, แคชเชียร์, และทีม Dev ที่เพิ่งเข้าโปรเจกต์

---

## แอพนี้คืออะไร?

**EasyShop POS** คือแอพขายหน้าร้าน (Point of Sale) บนมือถือ สำหรับร้านค้าไทย
ลูกค้าสั่งอะไร → แคชเชียร์กดเพิ่มในตะกร้า → กด "ชำระเงิน" → แอพสร้าง QR PromptPay ให้ลูกค้าสแกนจ่ายได้เลย ไม่ต้องใช้เครื่อง EDC

**ใช้กับอะไรได้บ้าง:** iPhone / Android / iPad

---

## ใครใช้แอพนี้?

| บทบาท | ทำอะไรได้ |
|--------|-----------|
| **เจ้าของร้าน (Owner)** | จัดการสินค้า, ดูออเดอร์ทั้งหมด, แก้ราคา/สต๊อก |
| **แคชเชียร์ (Cashier)** | เปิดบิล, รับชำระเงิน QR, ดูประวัติออเดอร์ |

---

## หน้าจอในแอพมีอะไรบ้าง?

### 1. หน้า Login
- กรอก Email + รหัสผ่าน แล้วกด "เข้าสู่ระบบ"
- แต่ละร้านแยกข้อมูลกันอย่างสมบูรณ์ (ข้อมูลร้านคุณไม่รั่วไปที่อื่น)

---

### 2. หน้า POS หลัก (แท็บแรก — ไอคอน 🛒)
นี่คือหน้าหลักที่ใช้ตอนขายของ

**วิธีใช้:**
1. ด้านบนมีปุ่มหมวดหมู่ (เช่น เครื่องดื่ม, อาหาร, ของหวาน) — กดเพื่อกรอง
2. กล่องค้นหาด้านบน — พิมพ์ชื่อสินค้าหาได้เลย
3. กดการ์ดสินค้า → เพิ่มลงตะกร้าทันที
4. ปุ่มสีเขียวมุมขวาล่าง (รูปตะกร้า) → ไปหน้าตะกร้า

**สิ่งที่เห็นในการ์ดสินค้า:**
- รูปสินค้า
- ชื่อสินค้า
- ราคา (บาท)
- จำนวนสต๊อกคงเหลือ (แดง = หมด, เหลือง = ใกล้หมด, เขียว = ปกติ)

---

### 3. หน้าตะกร้า (กดปุ่มตะกร้าจาก POS)
หน้านี้ดูสรุปของที่เลือกก่อนชำระเงิน

**ทำอะไรได้:**
- เพิ่ม/ลดจำนวนสินค้าแต่ละรายการ
- ลบสินค้าออกจากตะกร้า
- กด "เพิ่มส่วนลด" → กรอก % ส่วนลด (0–100)
- ดูยอดรวม: ราคา → ส่วนลด → VAT 7% → **ยอดสุทธิ**
- กด **"ชำระเงิน QR"** → ไปหน้า QR

---

### 4. หน้า QR Payment (หน้าชำระเงิน)
หน้านี้เปิดขึ้นมาหลังกดชำระเงิน

**ขั้นตอน:**
1. แอพสร้าง QR PromptPay ให้อัตโนมัติ
2. ลูกค้าเปิดแอพธนาคาร → สแกน QR → โอนเงิน
3. แอพรอรับยืนยันจากระบบ (นับถอยหลัง 5 นาที)
4. ถ้าระบบแจ้งว่าจ่ายแล้ว → หน้าจอเปลี่ยนเป็น ✅ ชำระสำเร็จ
5. ถ้าลูกค้าจ่ายแล้วแต่ระบบยังไม่ตอบ → กด **"ยืนยันรับเงินแล้ว"** (manual)
6. ถ้าหมดเวลา → กด "ยกเลิก" แล้วเริ่มใหม่

---

### 5. หน้าออเดอร์ (แท็บ 2 — ไอคอนรายการ)
ดูประวัติการขายทั้งหมด

**ทำอะไรได้:**
- เห็นรายการออเดอร์ทุกใบ พร้อมสถานะ (รอ / สำเร็จ / ยกเลิก)
- กดออเดอร์ → เปิดรายละเอียด (รายการสินค้า, ราคา, วิธีชำระ)
- ดูว่าออเดอร์ยืนยันด้วย Auto หรือ Manual

---

### 6. หน้าสินค้า (แท็บ 3 — ไอคอนกล่อง)
จัดการสินค้าในร้าน (เจ้าของร้านใช้เป็นหลัก)

**ทำอะไรได้:**
- ค้นหาสินค้าด้วยชื่อ
- ดูรายการสินค้าทั้งหมด พร้อมราคา/สต๊อก/หมวดหมู่
- กดปุ่มดินสอ ✏️ → แก้ไขสินค้า (ชื่อ, ราคา, สต๊อก, หมวดหมู่)
- กดปุ่ม + มุมขวาล่าง → เพิ่มสินค้าใหม่

## วิธีติดตั้งและเปิดใช้งาน (สำหรับ Dev)

### ขั้นที่ 1 — ดาวน์โหลดโค้ดและติดตั้ง
```bash
git clone <repo-url>
cd QRForPay
npm install
```

### ขั้นที่ 2 — ตั้งค่า Supabase
สร้างไฟล์ `.env` ที่โฟลเดอร์หลัก แล้วกรอก:
```
EXPO_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=eyJ...
EXPO_PUBLIC_PROMPTPAY_ID=0812345678   ← เบอร์ PromptPay ร้านคุณ
```

### ขั้นที่ 3 — สร้างฐานข้อมูล
เข้า Supabase Dashboard → SQL Editor แล้วรันไฟล์ตามลำดับ:
1. `supabase/schema.sql` — สร้างตาราง
2. `supabase/rls_policies.sql` — ตั้งสิทธิ์ความปลอดภัย
3. `supabase/seed.sql` — ใส่ข้อมูลตัวอย่าง (10 สินค้า)

### ขั้นที่ 4 — รันแอพ
```bash
npx expo start
```
แล้วสแกน QR ด้วย Expo Go บนมือถือ

## ระบบฐานข้อมูล (อธิบายแบบชาวบ้าน)

แอพเก็บข้อมูลใน 7 "ตาราง" (เหมือนชีต Excel):

| ตาราง | เก็บอะไร |
|-------|----------|
| `shops` | ข้อมูลร้านค้า (ชื่อร้าน, เบอร์ PromptPay) |
| `profiles` | ข้อมูลพนักงาน (email, บทบาท owner/cashier) |
| `categories` | หมวดหมู่สินค้า (เครื่องดื่ม, อาหาร ฯลฯ) |
| `products` | สินค้า (ชื่อ, ราคา, สต๊อก, รูปภาพ) |
| `orders` | ออเดอร์ทุกใบ (วันที่, ยอดรวม, สถานะ) |
| `order_items` | รายการสินค้าในแต่ละออเดอร์ |
| `payments` | การชำระเงิน (QR payload, สถานะ success/pending) |

**ความปลอดภัย:** แต่ละร้านเห็นเฉพาะข้อมูลตัวเองเท่านั้น (ระบบ RLS)

## การคำนวณราคา

```
ราคาสินค้า (รวม VAT แล้ว)
  - ส่วนลด (ถ้ามี %)
  = ยอดหลังลด
  → แยก VAT 7% ออกมาแสดง (รวมอยู่ในราคาอยู่แล้ว)
  = ยอดที่ต้องจ่าย (Total)
```

ตัวอย่าง: สินค้า ฿107 → ลด 0% → VAT = ฿7 → Total = **฿107**

## ปัญหาที่พบบ่อย

| ปัญหา | วิธีแก้ |
|--------|---------|
| ภาพสินค้าไม่แสดง | รัน `supabase/migrations/update_product_images.sql` ใน SQL Editor |
| Login ไม่ได้ | ตรวจสอบ `.env` ว่า URL และ Key ถูกต้อง |
| QR ไม่ตอบสนองเมื่อจ่ายแล้ว | กดปุ่ม "ยืนยันรับเงินแล้ว" แทน (manual confirm) |
| แอพช้าหรือค้าง | ดึงข้อมูลใหม่ได้โดยลากลงจากด้านบนของรายการ (pull to refresh) |
